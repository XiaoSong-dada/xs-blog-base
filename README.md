# xs-blog-base

这是小宋博客的基础资料仓库，主要用于集中存放前端、后端相关的公共文档、设计说明、开发记录以及后续可沉淀的项目资料。

## 项目定位

`xs-blog-base` 不承担前端页面渲染或后端接口服务能力，它的职责更偏向于文档中台，主要用于：

- 存放前端项目相关文档。
- 存放后端项目相关文档。
- 统一沉淀开发过程中的说明材料、约定和设计文档。
- 为前后端协作提供一个独立、清晰的资料归档位置。

## 适用场景

这个仓库适合放置以下内容：

- 功能设计文档
- 接口说明文档
- 数据结构说明
- 部署文档
- 开发规范与约定
- 联调记录、问题排查记录
- 迭代计划或阶段性总结

## 目录结构

当前仓库目录如下：

```text
doc/
	service/   后端相关文档
	web/       前端相关文档
```

建议约定：

- `doc/web`：存放 `xs-blog-web` 对应的页面说明、组件设计、交互文档、构建与部署说明等。
- `doc/service`：存放 `xs-blog-service` 对应的接口文档、数据库设计、部署说明、迁移记录等。

## 与其他仓库的关系

在当前工作区中，项目职责可以大致分为三部分：

- `xs-blog-web`：博客前端项目。
- `xs-blog-service`：博客后端服务项目。
- `xs-blog-base`：前后端公共文档与资料仓库。

这种拆分方式有几个好处：

- 文档与代码分离，避免主业务仓库中堆积大量说明性文件。
- 前后端资料可以集中整理，便于查找和维护。
- 更适合后续持续补充架构文档、接口说明和部署经验。

## 使用建议

为了便于长期维护，建议在本仓库中保持统一的文档命名和分类方式，例如：

- 按模块拆分文档，而不是把所有内容堆在一个文件中。
- 文档标题尽量明确，例如“用户模块接口说明”“文章发布流程说明”。
- 对于会频繁变更的文档，标注更新时间和适用范围。
- 对于跨前后端的内容，可以单独新增专题文档统一说明。

## 维护说明

如果后续需要，可以在这个仓库继续补充以下内容：

- 项目总览文档
- 系统架构图
- API 汇总文档

## 总结

`xs-blog-base` 的核心目标是作为小宋博客项目的文档沉淀仓库，集中管理前端与后端的说明资料，让文档归档更清晰、协作更顺畅、后续维护更方便。

## 统一容器启动（主模块入口）

当前仓库已支持在主模块统一编排 `web + api + migrate + db + redis`。

### 快速启动

1. 在仓库根目录准备环境变量（建议从 `.env.example` 复制为 `.env` 后按本机环境修改）。
2. 在仓库根目录执行：

```bash
docker compose up --build
```

### 服务说明

- `db`：PostgreSQL（含初始化脚本）。
- `redis`：Redis（使用模块内配置文件）。
- `migrate`：后端 Alembic 迁移任务。
- `api`：FastAPI 服务。
- `web`：Vue 构建产物由 Nginx 托管，并反向代理 `/api` 与 `/static`。

### 访问地址

- 前端站点：http://localhost
- 后端接口：http://localhost:8000

说明：

- 后端子模块内的 `docker-compose.yml` 可用于后端单模块调试，但完整系统联调以主模块根目录的 `docker-compose.yml` 为准。
